Actions

Up 
get /publisher/{publisher}/process/{processInstanceId}/states/{state}/actions/{actionKey}
Get action associated to process with updated status (getAction)
Get action associated with Third+process+state, returns an array of actions (application/json)

Path parameters

processInstanceId (required)
Path Parameter — Process instance unique id
state (required)
Path Parameter — Name of state
actionKey (required)
Path Parameter — key of action (unique for state)

Return type

ActionStatus

Example data

Content-Type: application/json
{
  "lockAction" : true,
  "updateStateBeforeAction" : false,
  "hidden" : true,
  "buttonStyle" : "buttonClass",
  "label" : {
    "key" : "myCard.myAction.label",
    "parameters" : {
      "EN" : "Choose colour",
      "FR" : "Choisir une couleur"
    }
  }
}

Produces

This API call produces the following media types according to the Accept request header; the media type will be conveyed by the Content-Type response header.
  • application/json

Responses

OK ActionStatus

400

Submitted data are incorrect.

Authentication required

Action - Up

defines an action on the business process associated to the card
type
ActionEnum Action type
url (optional)
String Url of remote service entry point
lockAction (optional)
Boolean if true, action will be locked after clicked client-side
called (optional)
Boolean True if action has already been called at least once
updateStateBeforeAction (optional)
Boolean if true, client will attempt an update of ActionStatus before execution
hidden (optional)
Boolean This action is hidden from card and is only available to details;
buttonStyle (optional)
String css classes of action button
label
I18n button default label, appears as a tooltip

ActionEnum - Up

Action type >

  • EXTERNAL - Not defined (not implemented)
  • URL - The action is tied to a url which must conform the specification of 3rd Party actions (see reference manual)
  • JNLP - The action triggers a JNLP link (not implemented)
