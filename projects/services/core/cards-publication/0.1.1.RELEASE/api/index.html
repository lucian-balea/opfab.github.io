get /cardSubscription
fetch card operations (getCardSubscription)
fetch cards restricted to calling user. Fetched cards all come with empty data and details. requesting this end point opens an SSE connection

Query parameters

rangeStart (optional)
Query Parameter — start of time range to get published card
rangeEnd (optional)
Query Parameter — end of time range to get published card
notification (optional)
Query Parameter — If true, connection is kept for notification of new cards (not related to the specified range) default: false
clientId (optional)
Query Parameter — A unique id to identify client to allow for reconnection (an autogenerated UUID is the best solution)

Return type

array[CardOperation]

Example data

Content-Type: application/json
[ {
  "number" : 3,
  "publishDate" : 1546300800000,
  "type" : "DELETE",
  "cardIds" : [ 12345, 12346, 12347 ]
}, {
  "number" : 3,
  "publishDate" : 1546300800000,
  "type" : "DELETE",
  "cardIds" : [ 12345, 12346, 12347 ]
} ]

Recipient - Up

Recipient object defines rules for recipient computation
type (optional)
RecipientEnum Specifies the recipient type. Please note that BROADCAST and DEADEND do not expose any other properties
recipients (optional)
array[Recipient] Used by INTERSECT, UNION with multiple Recipients. Used by FAVORITE, RANDOM, WEIGHTED for single Recipient

Boolean If main users from the underlying recipients are to be propagated. Used by INTERSECT and UNION recipients.

RecipientEnum - Up

Recipient type >

  • BROADCAST - Card is sent to everybody
  • DEADEND - Card is sent to no one (mostly for testing purposes)
  • GROUP - Card is sent to every user belonging to a group (identity)
  • USER - Card is sent to a single user (identity)
  • UNION - Card is sent to users according to the union of a recipient list (recipients)
  • INTERSECT - Card is sent to users according to the intersection of a recipient list (recipients)
  • RANDOM - Card is sent to user as defined by underlying recipient (recipient) a random user is picked as the main recipient
  • WEIGHTED - Card is sent to users as defined by underlying recipient (recipient) a random user is picked as the main recipient. A favorite user (favorite) may be specified and is favored in the random choosing process.
  • FAVORITE - Card is sent to user as defined by underlying recipient (recipient), if favorite user (favorite) is in the computed users list it will be set as main recipient

SeverityEnum - Up

Severity of the card subject >

  • ALARM - The process instance behind the card is in critical condition
  • ACTION - The process instance behind the card is expecting an action from the user
  • QUESTION - The process instance behind the card needs information from the user
  • NOTIFICATION - Purely informational card
